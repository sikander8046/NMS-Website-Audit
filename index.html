Critical
The CMS Is a 17-Year-Old Proprietary Framework
Your site runs on the "Irongaze Framework" — a custom CMS built specifically for NMS in 2008. It was genuinely ahead of its time: you can edit content yourselves, the calendar module is custom-built for your workflow, you have an online registration system, and payments are integrated. That infrastructure still works.

The challenge is not capability — it is age. The Irongaze Framework is 17 years old. It predates smartphones, responsive design, WebP images, Core Web Vitals, and modern booking integrations. It cannot be extended to support these without rebuilding from scratch. The proprietary architecture also means only Irongaze can work on it — there is no open ecosystem of developers who can maintain or enhance it. What was innovative in 2008 is now a ceiling on what the site can become.
Fix: Rebuild on Next.js with a headless CMS (Sanity or Contentful). You retain full self-editing capability — exactly as you have now — but gain a modern, open platform that any developer can maintain, extend, and improve. The transition preserves everything that currently works and removes the ceiling it creates.
Warning
Calendar Directory Is Blocked by robots.txt
The calendar section at /calendar/nms-schedule is blocked from search engine crawling by the site's robots.txt file. This means Google cannot index the trip schedule pages — a significant missed SEO opportunity. Users searching for "Mt. Baker guided climb June 2026" or "AIARE Level 1 course Washington" would not find the calendar, even if it contains exactly that information.
Fix: Remove the /calendar/ path from robots.txt disallow rules. The calendar is a custom Irongaze module — its separate URL structure (/calendar/ vs /content/main/) is by design, not a technical problem. The problem is solely the robots.txt block. Removing that single line immediately makes all trip dates indexable by Google. Add Schema.org Event markup to every dated program listing for rich snippet eligibility.

Calendar System Analysis

The trip schedule — the most important booking tool on the site — is effectively invisible.

📅

Your Calendar — What It Is and What's Wrong

The Calendar nav item links to /calendar/nms-schedule — a custom module Irongaze built specifically for NMS in 2008, which is why it has a different URL structure from the rest of the site. This was innovative when built. It lets you publish trip dates that potential clients can view. The critical problem: this page is actively blocked from search engine crawling by your site's robots.txt file, making all your trip dates completely invisible to Google.

The calendar URL pattern (/calendar/) differs from all other page URLs — indicating a separate, possibly embedded system
Blocked by robots.txt — Google cannot index any trip dates, meaning no organic search traffic from date-specific queries
No integration with booking — users can see dates but cannot reserve or pay directly from the calendar
No availability indicators — no way to see how many spots remain on any given trip
Not linked from individual trip pages — a user reading about the Denali climb has no direct path to see upcoming Denali dates
No mobile-optimised calendar interface — table-based layout breaks on phones
Critical
The Calendar Is Invisible to Google — and Disconnected from Booking
A prospective client searching for "guided Mt. Rainier climb August 2026" or "backcountry ski course Washington winter 2027" would never find NMS's calendar — it is hidden from Google. Even if they land on the site through other means, the calendar is siloed: it shows dates but offers no path to reserve a spot. The user must navigate away, find the Registration page, read instructions, then email or call. Every step loses a percentage of potential clients.
Fix: Replace with FareHarbor or Peek Pro — both are built specifically for adventure guiding companies. These integrate with the website, show real availability, allow online reservations and deposits, and generate Schema.org Event markup automatically. Every dated program should be bookable in under 3 clicks.
Context
How Competitors Handle Booking
Leading US guide services — Alpine Ascents International, International Mountain Guides, American Alpine Institute — all use FareHarbor or RezDy for integrated online booking. Clients can see available dates, remaining spots, and complete a deposit in one session. NMS's current phone/email-only booking model works because of their strong word-of-mouth reputation and 70% advance bookings — but it creates an invisible ceiling on new client acquisition from web discovery.
Recommendation: Implement FareHarbor ($0 setup, commission-based model) as the primary booking layer. Integrate availability widgets directly on each trip page and the calendar page. Remove robots.txt block on the /calendar/ path.

Visual Design & Brand

Patagonia-sponsored guides. A .gif logo. The contrast is stunning.

Critical
The Logo Is a .GIF File from 1989's Format Standard
/images/logo.gif — this file format was introduced in 1989. On any modern display (which means virtually every laptop, phone, and monitor sold in the last decade), this logo appears pixelated and blurry. For a guide service charging $5,000–$15,000 for international expeditions and wearing Patagonia sponsorship, this is the first impression. It signals: we have not updated this website since the early 2000s.
Fix: Redesign the logo as an SVG file — infinitely scalable, zero quality loss at any size. Design direction: a clean geometric mountain mark paired with a serif or condensed wordmark. The identity should feel like it belongs on the spine of an IFMGA certification document.
Critical
No Brand System — No Color Palette, No Typography, No Visual Identity
There is no defined color palette, no typographic hierarchy, no visual grid, and no consistent design language across the site. The site uses default browser link colors (blue underlines), generic system fonts at a single weight, and a layout that has not been intentionally designed. The Patagonia and Petzl logos in the footer are better designed than anything on the NMS site itself.
Fix: Establish a complete brand system. Proposed direction: deep glacier navy as primary (conveys authority, altitude, trust), ember orange as action color (warmth, energy, urgency), parchment white as background (expedition journal feel). Typography: Playfair Display for editorial headlines (prestige, depth), Outfit for body (clean, readable). Every page should feel like a premium mountain guide manual.
Critical
Homepage Has No Visual Hierarchy or Hero
The homepage opens with a single static mountain JPEG served within a narrow content column. There is no headline, no tagline, no subheadline, no CTA, and no visual hierarchy. A first-time visitor has no orientation: who are you, what do you offer, why should I trust you, what should I do next? All of these questions go unanswered in the first 5 seconds — which is how long most visitors decide whether to stay or leave.
Fix: Full-viewport cinematic hero with: a bold headline ("America's Most Certified Mountain Guides"), a one-sentence value proposition, IFMGA/AMGA certification badges, and two CTAs ("Explore Expeditions" primary, "Meet John & Olivia" secondary). The hero image should span 100% of the viewport and communicate the scale and beauty of what NMS offers.
Warning
Photography Is the #1 Sales Asset — and Is Criminally Underused
NMS has genuine expedition photography from Denali, the Cascades, the Alps, and beyond. The current site displays this photography as small constrained thumbnails inside a narrow content column, surrounded by navigation chrome and white space. Adventure travel is a visual purchase. Clients book when they feel the mountain calling them. The photography that should be doing that work is being served at the equivalent of passport photo size.
Fix: Full-bleed, edge-to-edge photography on every key page. Hero images that dominate the viewport. Parallax scroll effects on section transitions. Large-format photo galleries on trip pages with lightbox viewing. The photography is the most powerful conversion tool available — treat it that way.

Critical
5-Level Navigation Hierarchy — Unusable on Mobile
The main navigation has 7 top-level items, each expanding into 5–8 sub-items, some with further sub-sub-items. The Climbing section alone: Climbing → Alpine → Mountaineering Courses → 5-Day Mountaineering Course. That is 4 levels deep for a single trip. On mobile, this collapses to a hamburger menu requiring multiple taps and providing no visual hierarchy. There are 50+ trip pages with no filter, sort, or search system to help users narrow down what they want.
Fix: Flatten to 4 top-level items: Trips, Courses, About, Contact/Book. Add a smart filter on the Trips index: filter by Activity (Ski / Climb / Rock / Expedition), Difficulty, Season, Duration. Every trip findable in 2 clicks maximum. Add a persistent search bar.
Critical
Trip Pages Have No Booking Infrastructure
Every trip page — Denali, Matterhorn, Haute Route — has excellent content but zero conversion architecture. There is no price visible in the first screen, no sticky CTA as the user scrolls, no date selector, no availability indicator, no guide profile showing who leads this specific trip. The registration process requires leaving the page, finding the Register nav item, reading text instructions, then emailing or calling. This multi-step friction eliminates impulsive or international bookings entirely.
Fix: Add a sticky booking sidebar to all trip pages: Price prominently at top, upcoming available dates, spots remaining, "Reserve Your Spot" primary CTA, "Ask a Question" secondary. This sidebar follows the user as they scroll through the itinerary — the booking opportunity never disappears from view.
Warning
"2021 Set Date Programs" Is Still Live in Resources
The Resources navigation section contains a link to "2021 Set Date Programs" — a page that is 4 years out of date. This signals to any visitor that the site is not actively maintained. It also creates potential confusion when clients check for current program availability. Small detail, large impression damage.
Immediate fix: Remove this link from the navigation today. It can be done without a full rebuild.
